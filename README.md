# Prompt Temperature Demo

## Objective

This project demonstrates how the **temperature** parameter affects responses generated by a GPT-style language model using the OpenRouter API.

## Technologies Used

- Python 3
- OpenAI Python SDK
- OpenRouter API
- GPT-OSS-20B (Free Model)
- python-dotenv

## Prompt Used

```
Explain Artificial Intelligence in simple words.
```

## Code Execution

The program sends the same prompt to the model using four different temperature values:

- 0.0
- 0.3
- 0.7
- 1.0

and prints the generated response for each.

# Effect of Temperature

| Temperature | Response Characteristics |
|-------------|--------------------------|
| **0.0** | Produces highly accurate, deterministic, and consistent responses. The output is factual and almost identical every time. |
| **0.3** | Produces focused responses with slight variation while maintaining accuracy. |
| **0.7** | Produces balanced responses that are both informative and creative. Suitable for general-purpose applications. |
| **1.0** | Produces the most creative and diverse responses. The output may vary significantly between executions and can be more imaginative. |

---

# Conclusion

The experiment demonstrates that the **temperature** parameter controls the randomness of the model's responses.

- Lower temperature values generate predictable and consistent outputs.
- Medium temperature values provide a balance between accuracy and creativity.
- Higher temperature values encourage more creative and diverse responses.

Therefore, selecting an appropriate temperature depends on the application:
- **Low temperature** for factual tasks such as question answering or summarization.
- **Medium temperature** for general conversation.
- **High temperature** for brainstorming, storytelling, and creative writing.
